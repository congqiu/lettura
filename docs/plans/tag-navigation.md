# Plan: 标签导航和筛选改进

## 问题分析

当前标签功能存在以下问题：

1. **侧边栏没有标签列表** — AppSidebar 只有"全部/未读/已归档/星标/已删除"分类，没有标签导航。用户无法快速按标签浏览文章。
2. **文章卡片不显示标签** — EntryListPage 的文章卡片只显示标题、域名、阅读时间，看不到标签。用户无法在列表中识别文章属于哪些标签。
3. **EntryTags 组件有 bug** — `handleAddTag` 中 `setNewTag('')` 调用时机不对（在异步操作前），且缺少加载状态和错误提示。
4. **标签筛选只能通过 URL 参数** — 没有可视化的标签筛选 UI，用户需要手动拼 URL 参数。

## 改进方案

### 1. 侧边栏添加标签列表

**文件**: `web/src/components/layout/AppSidebar.tsx`

- 在"星标"和"已删除"之间添加"标签"分组
- 调用 `GET /api/v1/tags` 获取用户所有标签
- 每个标签显示名称和文章数量
- 点击标签导航到 `/?tag=<label>`
- 最多显示 10 个标签，超出部分显示"查看全部"链接
- 使用 `useQuery` 缓存标签数据

### 2. 文章卡片显示标签

**文件**: `web/src/pages/EntryListPage.tsx` 或 EntryCard 组件

- 在每张文章卡片底部显示标签（使用 TagBadge 组件）
- 最多显示 3 个标签，超出显示 "+N"
- 标签可点击，点击后筛选该标签

### 3. 修复 EntryTags 组件

**文件**: `web/src/components/EntryTags.tsx`

- 修复 `setNewTag('')` 时机：移到 API 调用成功后
- 添加加载状态（添加标签时按钮显示 spinner）
- 添加错误提示（添加失败时显示 toast）
- 防止重复提交

### 4. 标签筛选 UI

**文件**: `web/src/pages/EntryListPage.tsx`

- 当 URL 有 `?tag=xxx` 时，在列表顶部显示筛选标签（TagBadge + 关闭按钮）
- 点击关闭按钮清除标签筛选

## 实施步骤

1. 修复 EntryTags 组件 bug
2. 侧边栏添加标签列表
3. 文章卡片显示标签
4. 标签筛选 UI（筛选指示器）

## 不做的事

- 不做标签管理页面（后续独立实现）
- 不做标签自动补全（后续独立实现）
- 不做标签云/统计图表
- 不修改后端 API
